# Invariant x Artifact Consumer Matrix

## Purpose

This matrix tracks cybernetic invariants across the artifacts that must create,
preserve, check, review, compile, or test them.

A rule is not considered maintained when it only appears in prose. For every
new or changed invariant, update the row that identifies:

- where the rule is stated;
- where the artifact field or template captures it;
- which deterministic guard rejects missing or invalid structure;
- which review dimension judges semantic adequacy;
- which downstream consumer or compiler preserves it;
- which eval or test prevents regression.

For non-applicable consumers, record `N/A: <reason>` and leave no blank cells.
Deterministic guards should check structure and explicit values;
semantic sufficiency belongs to control review.

## Maintenance Checklist

When adding or changing an invariant:

1. Add or update one matrix row in the same change.
2. Fill every consumer column, using `N/A: <reason>` only when the consumer truly does not apply.
3. Add at least one regression test or eval that covers a negative case.
4. If the invariant affects runtime behavior, verify the runtime compiler or downstream script preserves it.
5. Run `python3 -m unittest tests.skills.test_invariant_consumer_matrix` before committing.
